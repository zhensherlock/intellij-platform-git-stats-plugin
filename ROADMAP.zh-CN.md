# GitStats 路线图

## 接下来

### 仓库与分支范围

- [ ] 通过 IntelliJ VCS/Git 仓库 API 检测 Git 根目录。
- [ ] 当项目存在多个 Git 根目录时，增加仓库选择器。
- [ ] 增加分支范围控制：当前分支、全部本地分支、指定分支或自定义修订范围。
- [ ] 在汇总和导出结果中包含当前仓库与分支范围。
- [ ] 对没有 Git 根目录的项目隐藏或弱化工具窗口状态。

### 作者明细

- [ ] 双击作者行时打开明细面板或二级表格。
- [ ] 展示每个作者的提交、日期、哈希、增删行数和修改文件。
- [ ] 增加在 IDE Git Log 中打开提交、打开变更文件的操作。
- [ ] 支持单独复制或导出作者明细行。

### 作者身份规则

- [ ] 同时解析作者邮箱和作者名称。
- [ ] 支持可用的 Git mailmap。
- [ ] 增加本地别名规则，用于合并同一作者的多个身份。
- [ ] 增加可选的机器人或服务账号排除规则。
- [ ] 在明细中展示被合并的身份，让聚合结果可追溯。

### 设置体验

- [ ] 为排除路径增加项目相对路径选择器。
- [ ] 校验排除路径，并高亮重复路径或不属于当前仓库的路径。
- [ ] 提供常见排除预设，例如构建产物、生成源码、第三方依赖和锁文件。
- [ ] 从 `.gitignore` 导入排除建议。
- [ ] 增加保存视图，复用常见的仓库、分支范围、日期范围、统计模式和排除路径组合。

## 后续

### 指标与维度拆分

- [ ] 增加合并提交、生成文件、二进制文件和重命名文件的统计开关。
- [ ] 区分修改文件事件和唯一修改文件数。
- [ ] 增加净增行数和代码 churn 指标。
- [ ] 增加按扩展名、目录或模块拆分的统计。
- [ ] 增加改动最多文件和高 churn 文件列表。
- [ ] 为 JVM 项目增加可选的包名或 source root 分组。

### 趋势与对比

- [ ] 增加提交数、添加行数、删除行数和修改文件数的日/周趋势视图。
- [ ] 增加与上一段等长周期的对比。
- [ ] 增加今天、昨天、上周、上月和自定义保存范围等时间预设。
- [ ] 在导出结果中包含趋势数据。
- [ ] 高亮相比上一周期异常增大的活动峰值。

### 报告与导出

- [ ] 增加 Markdown 导出，方便分享到 issue、pull request 或 release note。
- [ ] 在导出文件中包含过滤条件、统计模式、仓库、分支范围和生成时间。
- [ ] 增加复制选中行、全部可见行和仅汇总内容的操作。
- [ ] 增加轻量级快照历史，用于比较最近生成的报告。

### 大仓库性能

- [ ] 将刷新任务迁移到 IntelliJ 后台任务，并支持进度展示和取消。
- [ ] 在用户再次刷新、关闭工具窗口或修改过滤条件时取消旧任务。
- [ ] 按仓库、分支范围、日期范围、统计模式和排除路径缓存近期结果。
- [ ] 在可行时流式解析结果，让 UI 可以渐进更新。
- [ ] 为大型 Git 日志输出增加基准测试或 fixture 性能测试。

### 可视化

- [ ] 增加活动趋势图。
- [ ] 增加作者提交数和代码 churn 对比图。
- [ ] 增加目录或文件类型拆分图。
- [ ] 保持图表可选，并以表格优先，确保工具仍然快速且符合 IDE 原生体验。

### 质量与兼容性

- [ ] 随功能名称变化同步 README、Marketplace 文案、截图和产品内标签。
- [ ] 为分支/修订范围和 mailmap 选项增加命令构造测试。
- [ ] 为重命名、二进制文件、合并提交、带分隔符的作者名称和特殊路径增加解析 fixture。
- [ ] 在仓库选择功能存在后，增加多 Git 根目录的服务层测试。
- [ ] 为明细操作和设置校验增加聚焦的 Swing 测试。
- [ ] 发布前保持 `check`、`buildPlugin` 和 `verifyPlugin` 在配置的 IDE 范围内通过。
